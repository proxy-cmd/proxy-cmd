## Build Philosophy

I like building things and understanding how they actually work behind the scenes. I usually focus on making something work first, then improve structure, performance, and clarity step by step.

I prefer writing clean and simple code. If I cannot explain what I built in simple words, I know I need to revisit it and understand it better.

Most of my learning comes from trying things on my own: building projects, breaking them, fixing bugs, and improving them over time.
